# GCP 合规判定助手（中国 GCP 2026 修订版）

本技能把中国《药物临床试验质量管理规范》(2026年修订，**2026年9月1日起施行**，6章54条) 作为权威依据，帮助临床试验从业者把一个**具体实操问题**映射到**具体条款**，并给出明确的**合规判定**和**可执行建议**。

回答对象是行业专家。标准是：**条款级精确、可追溯、可落地**。不要泛泛而谈，不要只讲原则。

> 完整条款原文在 `references/gcp-full-text.md`。**做判定前必须先打开该文件核对相关条款的确切措辞**，引用要逐字、并标注条款号。本 SKILL.md 提供的是定位索引与判定方法，不能替代原文核对。

---

## 工作流程（每个问题都按此执行）

1. **拆解操作**：从问题中提取"谁、在试验哪个环节、做了/打算做什么动作、是否已发生"。从业者常省略主语和时态，必要时先确认是"已经做了"还是"打算做"。
2. **定位条款**：用下方"主题 → 条款定位索引"找到 1 个核心条款 + 若干关联条款。一个实操问题常牵动多条（如知情同意补签同时涉及第二十七条与方案偏离第二十三条）。
3. **核对原文**：打开 `references/gcp-full-text.md`，读到相关条款的**确切文字**。不要凭记忆复述条款内容。
4. **判定合规**：按下方"判定标尺"给出 **符合 / 违背 / 视情形而定** 三态之一，并说清触发判定的关键事实。
5. **给整改与建议**：违背→给纠正措施 + 应否作为偏离/不依从上报、向谁报、时限；视情形→列出决定走向的关键变量；符合→指出仍需留痕/可优化处。
6. **标注上报与时限触发**：凡涉及 SAE/SUSAR、重要方案偏离、严重持续不依从、安全性问题暂停终止的，明确"报谁 + 时限"。

## 判定标尺（如何给出三态结论）

- **违背 GCP**：操作直接抵触条款的强制性要求（条款中"应当""不得""必须""至少""立即""事先"等）。例：使用未经伦理批准的旧版 ICF 入组（违第二十七条（一））；研究者获知死亡 SAE 未立即报告（违第二十六条（二））；BE 留样返还申办者（违第二十八条（二））。
- **视情形而定**：合规与否取决于未在问题中给出的关键事实。**必须把决定走向的变量列清楚**，而不是含糊其辞。例："偏离方案是否违规"取决于是否记录、是否属重要偏离、是否为消除紧急危害、是否已报伦理与申办者（第二十三条）。
- **符合 GCP**：操作满足条款要求。即便符合，也应提示**留痕/可追溯**义务——GCP 的隐含红线是"没有记录=没有发生"。

**关键原则（贯穿所有判定）**
- 试验参与者(受试者)的**权益与安全是首要**，优先于科学与社会获益（第四条、第五条）。
- **质量源于设计 + 基于风险**：措施应与风险相称，不是一刀切（第六条、第四十条、第四十二条）。
- **可追溯/留痕**：记录须满足 ALCOA——可归因、易读、同时、原始、准确、完整；修改不得掩盖原始记录并须记录理由（第二十九条（二））。
- **最终责任不可外包**：申办者对委托活动负最终责任（第三十二条、第三十六条）；主要研究者是现场最终责任人（第十九条），其决策/关键确认/正式报告原则上不得授权（第二十条）。
- **利益冲突回避**（第十二条）。

### 一、总则与基本原则
- 适用范围（经批准/备案为研制药品的试验）：第二条
- 全过程定义（计划→启动→执行→记录→监督→评估→分析→报告）：第三条
- 赫尔辛基宣言/受试者权益优先：第四条；风险获益权衡：第五条
- 质量源于设计：第六条；方案须经伦理批准方可执行、修订须再批准：第七条
- 人员资质/培训/经验、角色职责留痕、医学决策由临床医生：第八条、第十九条、第二十五条
- 资料记录与个人信息保护：第九条
- 利益冲突回避：第十二条；新技术新方法：第十三条

### 二、伦理审查委员会
- 伦理职责/审查文件清单：第十四条；特殊/弱势人群审查：第十四条（二）
- 不得强迫利诱、不得要求放弃权益/免责条款：第十四条（三）
- 补偿合理性审查：第十四条（四）
- 重点及时审查事项（SAE、紧急偏离、严重持续不依从、新信息）：第十四条（五）
- **跟踪审查频率 ≤ 12 个月**：第十四条（六）
- 审查意见类型/书面意见与版本：第十四条（七）；暂停终止权：第十四条（八）
- **伦理记录保存：注册用→上市后5年；未获批/非注册用→终止后5年**：第十六条

### 三、主要研究者与机构
- 机构建立质量管理体系：第十八条
- PI 现场最终责任、资质熟悉方案/手册：第十九条
- **授权与委托：机构外委托须事先获申办者同意；最终责任不可推卸；决策/关键确认/正式报告原则上不得授权**：第二十条
- 完成试验必要条件（时间/入组/设施/监管研究人员）：第二十一条
- 与伦理沟通（启动前获批准、过程中报告、执行审查意见）：第二十二条
- **遵守方案与方案偏离**：第二十三条
  - 偏离须记录；重要偏离须解释+纠正预防+报伦理与申办者：第二十三条（一）
  - 为消除紧急危害可先偏离、随后及时报伦理与申办者并说明理由：第二十三条（二）
  - 揭盲：按方案；意外破盲/紧急揭盲须立即记录并书面向申办者说明原因：第二十三条（三）
- 提前终止/暂停须通知受试者并随访、向非发起方等报告：第二十四条
- **研究者安全性报告**：第二十六条
  - 按方案时限报告 AE/异常检查：第二十六条（一）
  - **除方案规定无需立即报告者外，获知 SAE 后立即书面报告申办者和伦理，随后及时随访报告**：第二十六条（二）
  - 死亡事件附加资料（尸检/最终医学报告）及时提供：第二十六条（三）
  - 收到 SUSAR/安全性信息须及时签阅、评估受试者治疗是否调整：第二十六条（四）
- **知情同意（核心，11 款）**：第二十七条
  - 充分告知+知情同意并记录、用最新批准版 ICF：第二十七条（一）
  - 出现影响参加意愿的新信息须及时告知，必要时重新签署：第二十七条（二）（亦见第四十四条（一））
  - 不得强迫利诱：第二十七条（三）
  - 通俗易懂语言：第二十七条（四）
  - 给予充分时间与机会、答疑：第二十七条（五）
  - **受试者与执行者分别签名注明日期、非本人须注明关系、病历记录知情时间与人员**：第二十七条（六）
  - 缺乏阅读能力须公正见证人全程见证：第二十七条（七）
  - 受试者须获已签字 ICF 原件及后续更新版：第二十七条（八）
  - 无民事行为能力人/限制民事行为能力人知情主体：第二十七条（九）
  - 紧急情况无法获知情同意的入组（须方案与文件事先经伦理批准载明）：第二十七条（十）
  - 无预期获益的试验原则上须本人签署：第二十七条（十一）
- 试验用药品现场管理责任、BE 留样：第二十八条
  - 专人管理、接收/贮存/分发/使用/回收/退还/销毁记录：第二十八条（一）
  - **BE 留样：随机抽取，至少保存至上市后2年；可委托独立第三方，但不得返还申办者或其利益相关方**：第二十八条（二）
- **记录与报告 / 源数据**：第二十九条
  - 监督数据采集、保证可靠：第二十九条（一）
  - **源记录须满足 ALCOA；修改可追溯、不掩盖原始记录、记录修改理由；以患者为受试者须载入门诊/住院病历系统**：第二十九条（二）
  - **必备记录保存：注册用→上市后5年；未获批/非注册用→终止后5年**：第二十九条（三）
  - 数据处理避免非法/未授权操作、保障安全：第二十九条（四）
  - 必备记录所有权转移合规：第二十九条（五）
- 接受监查/稽查/检查并配合提供记录：第三十一条

### 四、申办者
- 申办者为最终责任人、受试者保护与数据可靠为基本考虑：第三十二条
- 方案设计基于安全有效数据、纳入 QbD：第三十三条
- 团队/医学人员/沟通渠道：第三十四条
- 选择合格 PI 与机构：第三十五条
- **委托服务供应商(CRO)**：第三十六条
  - 委托须监督管理、承担最终责任；**转包须事先书面同意**：第三十六条（一）
  - 试验前与各方签合同、明确角色责任权利义务与利益冲突、经费合理：第三十六条（二）
  - 对申办者的要求适用于承担相应任务的服务供应商：第三十六条（三）
- 实验室资质、**禁止与伦理批准方案无关的生物样品检测(如基因检测)**、剩余样品说明：第三十七条
- 试验前提交资料获许可/BE备案、执行伦理意见：第三十八条
- 制定并更新方案/手册/知情材料、提供最新文件：第三十九条
- 基于风险的全过程质量管理与监督、措施与风险相称：第四十条
- 质量保证与质量控制（SOP、基于风险、监查与数据管理为主要 QC）：第四十一条
- 风险管理（识别/评估/预设可接受范围/记录沟通/定期审查/报告）：第四十二条
- **依从性管理**：第四十三条
  - 对不遵守方案/SOP/规范者采取纠正：第四十三条（一）
  - 显著影响权益安全/数据可靠的不依从→分析根因+纠正预防+及时书面报伦理：第四十三条（二）
  - **严重、持续不依从→考虑终止 PI/机构/供应商参加、书面报伦理、降低影响；严重违规可追责并报药监**：第四十三条（三）
- **安全性评估与报告**：第四十四条
  - 审阅评估安全信息、新信息及时通知 PI/机构/伦理并更新文件：第四十四条（一）
  - 收到安全信息立即分析评估（严重性/相关性/是否预期）：第四十四条（二）
  - **SUSAR/严重安全性风险信息快速报告 CDE；向 PI/伦理递交方式与紧迫性相称；紧急安全问题按快速报告时限报伦理与 PI**：第四十四条（三）
  - **定期向 CDE 提交药物研发期间安全性更新报告(DSUR)**：第四十四条（四）
  - 发现安全问题→及时调整方案/暂停或终止并报 CDE：第四十四条（五）
- **免费提供 IMP、支付相关医学检测费、保证补偿或赔偿**：第四十五条
- 试验用药品制备/供应/管理：第四十六条
  - GMP 条件生产放行、标签标明"仅用于临床试验"/盲态：第四十六条（一）
  - 规定贮存运输条件/有效期/用法、提供书面说明、保留记录、疫苗另符规定：第四十六条（二）
  - 获伦理批准与药监许可/备案后及时提供 IMP：第四十六条（三）
  - 供应管理规程、回收/未使用药品返还申办者或授权处置、全过程计数准确：第四十六条（四）
  - **盲法须建立紧急揭盲规程与机制**：第四十六条（五）
  - 保证稳定性、仅在有效期内提供、留样：第四十六条（六）
- 盲态维持与防止/识别破盲：第四十七条
- **数据治理责任（申办者）**：第四十八条
  - 电子数据系统满足计算机化系统要求：第四十八条（一）
  - **申办者不得擅自更改 PI/研究人员/受试者录入的数据；除非有正当理由、事先获 PI 书面同意并记录**：第四十八条（二）
  - 用受试者鉴认代码识别数据、揭盲后提供治疗信息：第四十八条（三）
  - 统计分析计划与质量管理：第四十八条（四）
  - 按监管要求保存必备记录并书面告知各方：第四十八条（五）
- 明确源记录查阅权限：第四十九条
  - 方案/合同中约定监查/稽查/伦理/药监可**直接查阅源记录**：第四十九条（一）
  - **每位受试者须书面同意上述各方直接查阅源记录**：第四十九条（二）
- 申办者变更/暂停/提前终止须按规定获批并书面告知各方说明理由：第五十条

### 五、数据治理
- 各方在职责范围内承担数据治理责任、贯穿数据全生命周期：第五十一条
  - 数据须附元数据(含稽查轨迹)：第五十一条（一）
  - 各方应用/评估/访问/管理元数据、制定审核规程：第五十一条（二）
  - 数据更正流程、及时更正并保证可追溯：第五十一条（三）
  - 系统间传输数据(含元数据)可靠/可追溯/安全、防丢失篡改：第五十一条（四）
  - 期中/最终分析数据按预定程序最终确认、遵循 SAP 并记录：第五十一条（五）
- 盲态完整性与防意外破盲偏倚：第五十二条
- **计算机化系统**：第五十三条
  - SOP、明确各方职责、人员培训：第五十三条（一）
  - 数据安全覆盖全生命周期、安全控制：第五十三条（二）
  - 系统验证、符合预期用途、始终处于验证有效状态：第五十三条（三）
  - 问题记录评估管理、定期审查识别系统性问题：第五十三条（四）
  - **用户/权限管理与稽查轨迹、授权访问、电子签名须符合我国电子签名要求、权限与职责/盲态相称并留痕**：第五十三条（五）

### 六、附则（术语）
- 术语定义（受试者/主要研究者/其他潜在严重安全性风险信息/质量管理/质量管理体系）：第五十四条
- 其余术语参考 ICH E6(R3) 中文版术语表：第五十四条

## 数值与时限速查表

| 事项 | 要求 | 条款 |
|---|---|---|
| 伦理跟踪审查间隔 | ≤ 12 个月 | 第十四条（六） |
| 伦理记录 / 必备记录保存（注册用，药物获批） | 至少至**上市后 5 年** | 第十六条、第二十九条（三）、第四十八条（五） |
| 必备记录保存（未获批 / 非注册用） | 至少至**临床试验终止后 5 年** | 第十六条、第二十九条（三） |
| BE 试验留样 | 随机抽取，至少至**上市后 2 年**；不得返还申办者或其利益相关方 | 第二十八条（二） |
| SAE（研究者→申办者+伦理） | 获知后**立即**书面报告，随后及时随访（方案另有规定的除外） | 第二十六条（二） |
| SUSAR（申办者→CDE） | **快速报告**；报 PI/伦理方式与紧迫性相称 | 第四十四条（三） |
| DSUR（申办者→CDE） | 定期提交，并通报 PI 与伦理 | 第四十四条（四） |
| 源数据标准 | ALCOA：可归因/易读/同时/原始/准确/完整 | 第二十九条（二） |

## 高频实操判定示例

**示例 1**
操作：CRC 发现某受试者上周的知情同意书漏签日期，今天让受试者补填了上周的日期。
判定：**违背**。第二十七条（六）要求签名并**注明日期**、病历记录知情同意的具体时间与人员；倒填日期破坏同时性(ALCOA 的"同时/原始"，第二十九条（二))，构成数据可靠性问题。整改：不得倒填；按真实日期记录补签并书面说明经过，评估是否构成方案偏离/不依从需按第二十三条、第四十三条上报；后续核对知情同意是否在任何研究程序前完成。

**示例 2**
操作：为抢救一名突发严重过敏的受试者，研究者来不及联系伦理就调整了给药，事后想知道是否违规。
判定：**视情形而定（通常合规）**。第二十三条（二）允许为消除对受试者的紧急危害在未获伦理同意的情况下偏离方案，但须**及时向伦理和申办者报告并说明理由**。关键变量：是否事后及时报告、是否记录。只要补报与记录到位即合规。

**示例 3**
操作：申办者数据管理直接在 EDC 里把研究者填错的一个数值改了，没通知中心。
判定：**违背**。第四十八条（二）规定申办者不得更改 PI/研究人员/受试者录入的数据，除非有正当理由、**修改前获得 PI 书面同意并记录**。整改：撤销/留痕该修改，改走 query 流程由中心确认；核对稽查轨迹完整性（第五十一条（一）、第五十三条（五））。
